Arquitectura del Sistema Aeroportuario

Diseño Técnico, Modelado de Datos y Procesos Operativos

1. Modelo de Datos Normalizado

Resumen del Modelo:

El sistema implementa una arquitectura de base de datos relacional diseñada para soportar operaciones críticas de gestión aeroportuaria sin redundancias. Los puntos clave incluyen:

  • Normalización (3FN): Se han separado las entidades para asegurar que cada dato se almacene en un solo lugar, facilitando actualizaciones consistentes.
  • Integridad Referencial: Relaciones estrictas entre Users, Flights y Bookings mediante llaves foráneas para evitar registros huérfanos.
  • Seguridad de Datos: Almacenamiento segregado de secretos para 2FA y contraseñas hasheadas.
  • Gestión de Inventario: Control en tiempo real de la ocupación de asientos vinculado directamente a la tabla de vuelos.

2. Diagramas de Secuencia

Resumen de Procesos:

Los diagramas de secuencia ilustran cómo fluye la información entre las capas de la aplicación (Frontend, API, Base de Datos y Servicios Externos):

  • Flujo de Autenticación: Implementa una verificación en dos pasos (2FA), asegurando que el acceso solo se conceda tras validar credenciales y un código dinámico (TOTP).
  • Flujo de Reserva: Define un proceso asíncrono donde la solicitud del pasajero queda en espera (PENDING) hasta que un agente o el sistema de pagos la valide, disparando notificaciones automáticas en cada cambio de estado.

3. Interfaces vs Casos de Uso

Resumen de Interfaces:

Cada interfaz del frontend ha sido diseñada para responder a uno o más casos de uso de negocio específicos, garantizando una experiencia de usuario orientada a tareas:

  • Pasajeros: Interfaces simplificadas para autoservicio (comprar, buscar, consultar).
  • Personal (Agentes/Admin): Dashboards de alta densidad de información para toma de decisiones rápida y gestión de inventario.
  • Sistema: Capas de comunicación invisible (emails, push) que mantienen al usuario informado sin intervención manual.
